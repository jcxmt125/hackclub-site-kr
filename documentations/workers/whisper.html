배포 방법

우선, 클라우드플레어 계정을 준비해주세요.
대시보드의 workers/pages 메뉴로 향하세요.

파란색 만들기 ("create") 버튼을 누르세요.

Whisper 워커 템플릿을 찾아 누르세요. Workers AI 바인딩 생성을 위해 필수인 단계입니다.

워커의 이름을 짓고, 생성하세요. (*.workers.dev 도메인을 만들어 달라는 요청을 받을 수 있어요!)

바로 코드 편집 (edit code) 버튼을 누르거나, 워커의 정보를 확인하세요. 여기서, 코드 편집 버튼을 누르세요.

코드를 편집기에 붙여넣고, 배포 (deploy) 버튼을 누르세요. 사용할 준비가 되었을 거에요!
더 쉬운 사용을 위해 설정, 트리거 (triggers), 로 향해 커스텀 URL이나 경로 (paths)를 설정하세요.
Wrangler (CLI) 배포는, 클라우드플레어의 도움말을 참고하세요.

배포 완료를 위해 조금의 추가 설정이 필요합니다.
[Your Unique Key] 부분에 당신만의 키를 채워넣어 주세요.
이 인증 방법은 "안전"하지 않아요! 큰 스케일로 배포하려면, 다른 방법을 이용해 주세요!

사용 방법

이 워커는 API 입니다.
GET 요청을 워커의 엔드포인트로 다음의 헤더와 함께 보내세요:

  • fileUrl: 텍스트로 변환하려는 오디오 파일의 URL
  • authKey: 미리 설정한 키
JSON 응답을 받을 거에요.
사용할 수 있는 파이썬 테스트 스크립트도 있습니다.
주의: 디스코드 CDN은 클라우드플레어 워커 플랫폼을 차단합니다.
